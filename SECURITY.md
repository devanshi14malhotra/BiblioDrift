# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability in BiblioDrift, please report it responsibly.

**Please do NOT open a public GitHub issue for security vulnerabilities.**

Instead, please report via one of the following methods:

1. **GitHub Security Advisories** — Use the "Report a vulnerability" feature on the Security tab of this repository.
2. **Email** — Send details to the maintainers (see [CONTRIBUTING.md](CONTRIBUTING.md) for contact info).

### What to include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

### Response timeline

- **Acknowledgment:** Within 48 hours
- **Initial assessment:** Within 7 days
- **Fix or mitigation:** As soon as practically possible

## Security Best Practices for Users

- Always use the latest version of BiblioDrift
- Review the permissions granted to any third-party integrations
- Do not expose API keys or credentials in configuration files
- Enable 2FA on your GitHub account

## Responsible Disclosure

We follow a coordinated disclosure process. If you report a valid vulnerability:

1. We will acknowledge receipt and begin investigation
2. We will keep you informed of progress
3. We will credit you in the advisory (unless you prefer anonymity)
4. We will not take legal action against good-faith security research
