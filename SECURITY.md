# Security Policy

## Supported Versions

We release patches for security vulnerabilities in the latest version only.

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability in BiblioDrift, please report it responsibly.

**Do not open a public issue.** Instead, email the maintainers at the address listed in the repository or use GitHub's private vulnerability reporting.

### What to include

- Description of the vulnerability
- Steps to reproduce
- Affected versions
- Potential impact

### Process

1. Submit your report via private channels
2. The team will acknowledge receipt within 5 business days
3. We will investigate and provide an initial assessment within 10 business days
4. Once resolved, we will publish a security advisory

We appreciate responsible disclosure and will credit researchers who follow these guidelines.
